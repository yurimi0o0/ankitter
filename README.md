# Ankitter

Ankiの代わりではなく、Ankiで覚える前にSNSを眺める感覚で暗記項目を浴びるタイムラインです。
React等のフレームワークは使わず、素のHTML/CSS/JavaScriptのみで構築されており、GitHub Pagesにそのまま配置して動作します。サーバーは不要で、データはすべてブラウザのIndexedDBに保存されます。

## 使い方

1. `index.html` を静的ホスティング（GitHub Pagesなど）に置くか、ローカルでは `python3 -m http.server` 等の簡易サーバーで配信して開きます（`file://` では `type="module"` のスクリプトが動かないブラウザがあるため、簡易サーバー経由を推奨）。
2. 初回起動時にAnki等から書き出したTSV（タブ区切り）ファイルを選択し、各列を「問題・答え・コメント・タグ・無視」に割り当てます。
3. タイムラインが生成され、Twitter風のUIで問題と答えが流れてきます。
4. ♡（いいね）は「覚えた」の意味で、押すと約7日間はタイムラインに出にくくなります（既知の項目が薄れ、未習の項目が中心になります）。7日経つと通常のローテーションに戻ります。
5. 🔁（リツイート）は「すぐまた見たい」の意味で、約10投稿後にもう一度タイムラインへ挿入されます。
6. 🔖（保存）は「後で見返したい」の意味で、約24時間後にタイムライン上部付近へ再表示されます。
7. 各投稿には、過去にタイムラインへ流した回数（インプレッション、♡と🔖の間の📊）と、各投稿表示でリアクションした回数（表示ごとの♡・🔁・🔖リアクション数）が各アイコンの右に表示されます。
8. 設定（⚙️）からTSVの追加・列割り当て変更・アイコン変更・ダークモード・答えの表示方法（同時表示 / ぼかしてタップ）を変更できます。
9. 設定画面からJSON形式でのバックアップのエクスポート・インポートが可能です。

## 構成

```
index.html          エントリーポイント
css/style.css        スタイル（ライト/ダーク対応）
js/db.js              IndexedDBの薄いラッパー
js/repo.js            ドメインロジック（TSV/カード/コメント/いいね/RT/設定の永続化）
js/tsv.js             TSVパースとカード変換
js/feed.js            タイムラインの並び替えエンジン（シャッフル・いいね再浮上・RT再挿入）
js/render.js           投稿DOMの生成
js/importFlow.js       TSV読み込み・列割り当てUI
js/settingsPanel.js    設定パネルUI
js/backup.js           JSONエクスポート/インポート
js/app.js              アプリ全体の初期化・配線
```

責務ごとにファイルを分離しているため、たとえば「投稿の見た目を変える」は`render.js`と`style.css`、「並び替えロジックを変える」は`feed.js`だけを触れば済むようになっています。
